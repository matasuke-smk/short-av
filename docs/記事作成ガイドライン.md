# Short AV 記事作成ガイドライン

このドキュメントは、Short AVの記事を作成・編集する際の統一ルールをまとめたものです。

## 📋 基本的な書き方のルール

### 1. 段落構成
- **1段落最大3文まで**
- 各段落の間に空行を入れる
- 段落が長くなりすぎないよう注意

### 2. 文章の長さ
- **1文30-40文字以内**を目安とする
- 複雑な構文は避け、シンプルに
- 読みやすさを最優先

### 3. 見出しの配置
- **200-300文字ごとに小見出しを配置**
- 見出しの階層を適切に使い分ける
  - `##` メインセクション
  - `###` サブセクション
  - `####` 詳細項目

### 4. 専門用語の扱い
- 初出時に（）で簡潔な説明を追加
- 例：「夜間陰茎勃起（NPT）」「海綿体（スポンジのような組織）」

### 5. 記事の冒頭
- **最初の100文字で記事の要点が分かるようにする**
- 読者の興味を引く内容にする
- 何が得られるかを明確に示す

## 🎨 視覚的配慮のルール

### 1. 強調表現

#### 【】マークの使用
重要なセクションや注目してほしい部分に使用：

```markdown
【安全なサウナの入り方】

以下のポイントを守れば、安心して楽しめます。
```

#### ■マークの使用
リスト項目のカテゴリー分けに使用：

```markdown
■ メリット

- 効果が高く、永久脱毛が可能
- 施術回数が少なく済む

■ デメリット

- 痛みが強い
- 費用が比較的高い
```

### 2. 箇条書きのルール

#### 項目数
- **箇条書きは3-5項目程度にまとめる**
- 多すぎる場合は複数のグループに分割

#### 書式（重要）
**NG：コロン形式**
```markdown
- **張力**：500-800g（痛みを感じない程度）
- **時間**：1日1-2時間から開始
```

**OK：改行形式**
```markdown
- **張力**
  500-800g（痛みを感じない程度）
- **時間**
  1日1-2時間から開始
```

#### 説明文の追加
箇条書きの前に必ず説明文を入れる：

```markdown
【推奨される栄養素】

効果を高めるために、以下の栄養素を摂取しましょう。

- **L-アルギニン**
  血流改善（3g/日）
- **亜鉛**
  組織修復（15mg/日）
```

### 3. 数字の表記
- **半角数字で統一**
- 例：「5回」「10-20万円」「3ヶ月」

### 4. 改行の効果的な使用
- セクション間：空行2行
- 段落間：空行1行
- 箇条書き内：適度な空行でグループ分け

## 📱 スマホ閲覧への配慮

### 1. 表（テーブル）の扱い
- **横長の表は絶対に使用しない**
- 代わりにカード形式で表示
- app/articles/[slug]/page.tsx で自動変換される

### 2. 文章の構造
- 長文は避け、箇条書きを活用
- 見出しを多めに配置して区切る
- スクロールしやすい構造にする

### 3. 画像や図表
- 横幅に収まるサイズ
- スマホで見やすい解像度

## 🔄 既存記事の改善例

### Before（NG）
```markdown
### サウナの効果

サウナには様々な効果があります。血流が改善され、代謝が上がり、デトックス効果もあります。また、リラックス効果やストレス解消、免疫力向上なども期待できます。

**主な効果：**
- 血流改善：全身の血行が良くなる
- デトックス：汗とともに老廃物を排出
- ストレス解消：副交感神経が優位になる
```

```markdown
### サウナの効果

サウナには様々な健康効果があります。

特に男性の健康にとって重要なポイントをご紹介します。

【サウナがもたらす3つの効果】

- **血流改善**
  全身の血行が良くなり、疲労回復を促進

- **デトックス効果**
  汗とともに老廃物を排出し、体内をクリーンに

- **ストレス解消**
  副交感神経が優位になり、深いリラックス状態に
```

## 📝 補足事項

### Markdown記法
- 太字：`**テキスト**`
- 見出し：`##`、`###`、`####`
- 箇条書き：`-` または `1.`
- リンク：`[表示テキスト](URL)`

### 禁止事項
- 絵文字の過度な使用（このガイドラインは除く）
- 煽り文句や誇大表現
- 根拠のない情報の掲載
- 個人を特定できる情報
- 差別的・攻撃的な表現

### 推奨事項
- データや研究結果の引用
- 医学的・科学的根拠の提示
- 読者への配慮ある表現
- 実用的で役立つ情報

## 🔧 技術的な注意点

### ファイルの場所
- 記事データ：`lib/articles.ts`
- 記事ページ：`app/articles/[slug]/page.tsx`
- 記事一覧：`app/articles/page.tsx`

### 記事の追加方法
1. `lib/articles.ts`にarticleオブジェクトを追加
2. このガイドラインに従って内容を記述
3. ビルドして確認：`npm run build`
4. 問題なければコミット＆プッシュ

### カスタムレンダリング
`app/articles/[slug]/page.tsx`では以下の自動変換が実行されます：
- Markdownテーブル → カード形式
- `**テキスト**` → 太字の白色テキスト
- `[リンク](URL)` → 青色のリンク
- リスト記法 → スタイル付き箇条書き
